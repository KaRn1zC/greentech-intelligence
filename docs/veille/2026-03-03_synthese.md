## Faits Marquants de la Semaine

### 1. L'UE adopte la directive Omnibus simplifiant la CSRD

- **Source** : [Council of the European Union / Gibson Dunn](https://www.gibsondunn.com/omnibus-simplification-of-eu-sustainability-rules-csrd-and-csddd-enacted/)
- **Date** : 24 fevrier 2026 (impact cette semaine)
- **Resume** : Le Conseil de l'Union europeenne a adopte le texte final de la directive Omnibus qui simplifie les obligations de reporting de durabilite (CSRD) et de devoir de vigilance (CSDDD). Le seuil d'applicabilite passe a plus de 1000 employes et 450 millions d'euros de chiffre d'affaires, excluant les PME du perimetre. Les entreprises de la "vague 2" et "vague 3" voient leur echeance reportee a 2028 pour l'exercice 2027.
- **Pertinence projet** : 🟡 Modere — Le cadre reglementaire europeen en matiere de durabilite evolue vers la simplification, mais le projet GreenTech Intelligence reste pertinent pour aider les entreprises a classifier et surveiller les informations Green IT, meme si les obligations directes sont allegees.

### 2. ROCm 7.2 : AMD etend le support PyTorch sur Windows pour les GPU Radeon

- **Source** : [AMD Blog - The Road to ROCm on Radeon](https://www.amd.com/en/blogs/2025/the-road-to-rocm-on-radeon-for-windows-and-linux.html)
- **Date** : Mars 2026
- **Resume** : AMD publie ROCm 7.2 avec un support natif de PyTorch sur Windows pour les GPU Radeon RX 7000 (RDNA 3) et RX 9000 (RDNA 4), ainsi que les APU Ryzen AI. Le driver 26.1.1 est requis. Les modeles Flux.1, SD3.5, DeepSeek R1 Distill et Llama 3.2 sont officiellement valides. Le support passe du stade "preview" a un statut plus stable.
- **Pertinence projet** : 🔴 Majeur — Impact direct sur notre infrastructure. Le PC Fixe (RX 7900 XTX) et le PC Portable (Ryzen AI 9 HX 370) beneficient pleinement de cette mise a jour. La compatibilite PyTorch + ROCm 7.2 sur Windows valide notre choix architectural pour l'entrainement et l'inference des modeles DeBERTa et BART.

- **Source** : [MLflow Releases](https://mlflow.org/releases)
- **Date** : Fevrier 2026
- **Resume** : MLflow 3.10.0 introduit le support des organisations multi-workspace, l'evaluation multi-tours pour les agents conversationnels, le suivi automatique des couts d'inference LLM dans les traces, et un redesign de la barre de navigation separant GenAI et ML classique. La version 3.10.1 (5 mars) corrige des bugs mineurs.

La mise a jour vers 3.10.1 est recommandee.

### 5. Hugging Face AI Energy Score : notation energetique des modeles

- **Source** : [Hugging Face AI Energy Score](https://huggingface.github.io/AIEnergyScore/)
- **Date** : Actualise en continu, reconnu par le gouvernement francais lors du AI Action Summit 2025

### 6. Empreinte carbone de l'IA : entre 32 et 80 millions de tonnes de CO2 en 2025

- **Source** : [ScienceDirect - Carbon and Water Footprints of Data Centers](https://www.sciencedirect.com/science/article/pii/S2666389925002788)
- **Date** : 2025-2026
- **Resume** : Une etude academique estime l'empreinte carbone des systemes d'IA entre 32,6 et 79,7 millions de tonnes de CO2 en 2025, et l'empreinte hydrique entre 312 et 764 milliards de litres. Une feuille de route (Cornell) montre qu'un deploiement optimise pourrait reduire ces impacts de 73% (CO2) et 86% (eau).
- **Pertinence projet** : 🟢 Mineur (mais utile pour le contexte) — Ces chiffres renforcent la legitimite du projet GreenTech Intelligence. Ils peuvent etre cites dans la documentation et la soutenance pour justifier l'importance de la classification Green IT.

**Contexte technique**

Le support PyTorch sur GPU AMD via ROCm a longtemps ete un frein pour les developpeurs souhaitant eviter l'ecosysteme NVIDIA. La sortie de ROCm 7.2 marque une etape significative :

- **Couverture materielle elargie** : Support natif des RX 7000 (RDNA 3), RX 9000 (RDNA 4) et APU Ryzen AI sur Windows et Linux.
- **Modeles valides** : DeepSeek R1 Distill, Llama 3.2, Flux.1, SD3.5 — couvrant inference LLM et generation d'images.
- **Driver unifie** : Le driver 26.1.1 integre directement le support PyTorch, simplifiant l'installation.
- **Passage du preview a la production** : ROCm 6.4.4 etait un preview public, ROCm 7.x stabilise le support.

- **PC Portable** (Ryzen AI 9 HX 370) : Inference et tests rapides. Le support APU Ryzen AI ouvre la possibilite d'utiliser le NPU pour l'acceleration.

### Analyse 3 : LoRA/QLoRA — Optimisations recentes pour le fine-tuning sur GPU consumer

**Contexte technique**

Les techniques de Parameter-Efficient Fine-Tuning (PEFT) continuent d'evoluer rapidement :

- **Reduction memoire 10-20x** : QLoRA permet le fine-tuning de modeles 7B sur un RTX 4090 (24 Go VRAM), conservant 80-90% de la qualite du fine-tuning complet.
- **LoRAFusion** : De nouveaux kernels optimises peuvent s'appliquer directement aux poids 4-bit QLoRA, evitant la dequantification intermediaire pour les grands volumes de tokens.
- **Optimiseurs pages** : Amelioration du throughput de 25% (628 vs 500 tokens/s) sur GPU consumer.
- **bf16 vs fp16** : Les benchmarks sur RTX 4060 montrent que bf16 degrade l'efficacite par rapport a fp16 sur GPU consumer — un point important pour notre RX 7900 XTX.
- **Pattern enterprise** : La tendance est au modele de base unique avec des adaptateurs LoRA specialises echangeables selon la tache, plutot que des modeles distincts.

**Implications pour GreenTech Intelligence**

Notre strategie de fine-tuning DeBERTa-v3 pour la classification Green IT repose sur LoRA (module `ai/models/training.py`). Points d'attention :

1. **VRAM budget** : La RX 7900 XTX (24 Go) est confortable pour le fine-tuning LoRA de DeBERTa-v3-base (~86M params + adaptateurs). QLoRA n'est probablement pas necessaire mais reste une option pour experimenter avec DeBERTa-v3-large.
2. **Format de precision** : Privilegier fp16 sur notre GPU AMD plutot que bf16, conformement aux benchmarks recents.
3. **Architecture multi-adaptateurs** : Envisager plusieurs adaptateurs LoRA pour differentes sous-taches (classification binaire Green IT, classification multi-label par categorie, detection de pertinence).

**Actions recommandees** :
- [ ] Valider le fine-tuning LoRA de DeBERTa-v3-base en fp16 sur ROCm 7.2
- [ ] Benchmarker LoRA rank r=8, r=16, r=32 sur notre dataset
- [ ] Evaluer QLoRA pour DeBERTa-v3-large comme alternative
- [ ] Tester le pattern multi-adaptateurs pour la classification multi-criteres

**Ressources** :
- [LoRAFusion Paper](https://arxiv.org/html/2510.00206v1)
- [Profiling LoRA/QLoRA on Consumer GPUs](https://arxiv.org/abs/2509.12229)
- [Fine-Tuning Infrastructure Guide](https://introl.com/blog/fine-tuning-infrastructure-lora-qlora-peft-scale-guide-2025)
